Лабораторная работа 7
Тема: Работа с иерархическими и сетевыми структурами в реляционных БД на примере логистической сети
Цели работы
1.	Освоить рекурсивные обобщённые табличные выражения (WITH RECURSIVE).
2.	Научиться представлять графовые структуры (деревья, сети) в реляционной модели.
3.	Решать задачи поиска путей, уровней вложенности, построения деревьев подчинения и маршрутов доставки.
4.	Сравнить эффективность рекурсивных запросов с многократными JOIN (на выбор).
Структура базы данных 
Таблица logistics_nodes
Хранит точки логистической сети (склады, хабы, магазины, пункты выдачи).
CREATE TABLE logistics_nodes (
    node_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    node_type VARCHAR(20) CHECK (node_type IN ('warehouse', 'hub', 'store', 'pickup_point')),
    parent_hub_id INT REFERENCES logistics_nodes(node_id) -- иерархическая привязка к хаб/склад
);
Таблица routes
Хранит связи между узлами (направленные или ненаправленные, с весом).
sql
CREATE TABLE routes (
    route_id SERIAL PRIMARY KEY,
    from_node INT NOT NULL REFERENCES logistics_nodes(node_id),
    to_node INT NOT NULL REFERENCES logistics_nodes(node_id),
    distance_km NUMERIC(8,2) CHECK (distance_km > 0),
    travel_time_min INT CHECK (travel_time_min > 0),
    UNIQUE (from_node, to_node)
);

Задание 1
Вывести всех потомков заданного узла (например, для node_id = 1). Используйте WITH RECURSIVE.
Задание 2
Вывести полное дерево с отступами (уровнем вложенности). Добавьте уровень (level) и сортировку по имени.
Задание 3
Найти все возможные пути между двумя узлами, например, из node_id = 1 в node_id = 5. Вывести последовательность узлов и общее расстояние. Ограничить глубину рекурсии (например, не более 5 переходов), чтобы избежать бесконечных циклов (используйте условие WHERE NOT ... или проверку на повтор узлов).
Задание 4
